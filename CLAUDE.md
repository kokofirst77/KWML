# CLAUDE.md — KWLM Lab 프로젝트 가이드

> 이 파일은 Claude Code가 프로젝트를 이해하고 일관성 있게 개발하기 위한 기준 문서입니다.
> 모든 작업 전에 반드시 이 파일을 먼저 읽으세요.

---

## 1. 프로젝트 목적 및 주요 기능 요약

### 목적
중학교 2학년 과학 광합성 단원 수업에서 교사가 바이브 코딩(Vibe Coding) 방식으로
설계한 KWLM 기반 AI 교육 서비스입니다.

학생들이 KWLM(Know / Want / Learned / More) 4단계를 순서대로 탐구하며,
Claude AI와 1:1 대화를 통해 사전 오개념을 진단받고, 질문을 구조화하고,
배운 내용을 시각화하고, 심화 탐구 과제를 추천받습니다.

교사는 대시보드에서 학급 전체의 응답 현황과 오개념 빈도를 실시간으로 모니터링합니다.

### 주요 기능 요약

| 단계 | 기능 |
|---|---|
| 로그인 | 학생 이름·반·번호 입력, 교사 코드 입력으로 대시보드 접근 |
| 홈 | KWLM 4단계 진행 현황 카드 (완료 시 ✅ 표시) |
| K (사전 지식) | Claude AI 챗봇과 소크라테스식 대화 → 오개념 진단 |
| W (알고 싶은 것) | 질문 입력 → 학급 전체 질문 워드클라우드 실시간 공유 |
| L (배운 것) | 텍스트 요약 입력 → Claude AI가 광합성 SVG 시각화 카드 생성 |
| M (더 알고 싶은 것) | 심화 질문 입력 → Claude AI가 실생활 탐구 과제 제안 |
| 교사 대시보드 | 학생 응답 현황, 오개념 빈도, 질문 군집 모니터링 |

## 2. 기술 스택

| 영역 | 기술 | 비고 |
|---|---|---|
| 프론트엔드 | Vanilla HTML / CSS / JavaScript | 단일 파일 구조, 프레임워크 없음 |
| AI | Claude API (claude-sonnet-4-20250514) | 챗봇·시각화·탐구제안 |
| 데이터 저장 | localStorage (브라우저) | 서버 없이 교실 PC에서 동작 |
| 배포 | 로컬 파일 실행 (file://) 또는 GitHub Pages | 인터넷 연결 필요 (API 호출) |
| 스타일 | CSS Variables + Flexbox/Grid | 외부 라이브러리 최소화 |

> ⚠️ 서버, 데이터베이스, Node.js, Python 백엔드는 사용하지 않습니다.
> 교실 PC에서 브라우저만으로 실행되어야 합니다.

## 4. 개발 시 지켜야 할 규칙과 주의사항

### 4-1. 코드 작성 규칙

- **파일 분리 원칙**: 페이지별로 JS 파일을 분리하고, `index.html`은 최대한 가볍게 유지합니다.
- **상태 관리**: 모든 학생 데이터는 `localStorage`에 저장합니다. 키 이름은 `kwlmlab_` 접두사를 사용합니다. (예: `kwlmlab_student`, `kwlmlab_k_chat`)
- **Claude API 호출**: 반드시 `api/claude.js`의 공통 함수를 통해서만 호출합니다. 페이지 파일에서 직접 `fetch`로 API를 호출하지 않습니다.
- **에러 처리**: API 호출 실패 시 반드시 사용자에게 친절한 한국어 안내 메시지를 표시합니다.
- **주석**: 모든 함수에 한국어 주석을 작성합니다. 교사(비개발자)가 읽을 수 있어야 합니다.

### 4-2. UI/UX 규칙

- **색상 테마**: 광합성·식물 이미지에 맞춰 초록 계열(`#2d6a4f`, `#52b788`, `#d8f3dc`)을 메인 컬러로 사용합니다.
- **폰트**: 별도 설치 없이 시스템 기본 폰트(Noto Sans KR, Google Fonts CDN)를 사용합니다.
- **반응형**: 교실 PC(1280px 이상) 기준으로 설계합니다. 모바일 대응은 v1.0 이후로 미룹니다.
- **네비게이션**: 상단 메뉴바에 K·W·L·M 4단계가 항상 표시됩니다. 완료한 단계는 ✅, 현재 단계는 강조 표시합니다.
- **로딩 상태**: Claude API 응답 대기 중에는 반드시 로딩 스피너와 "AI가 생각하는 중..." 메시지를 표시합니다.

### 4-3. Claude API 프롬프트 규칙

- 모든 시스템 프롬프트는 `data/prompts.js`에서 관리합니다. 페이지 파일에 프롬프트를 하드코딩하지 않습니다.
- **K단계 프롬프트**: 소크라테스식 반문 방식. 절대 정답을 바로 알려주지 않습니다.
- **W단계 프롬프트**: 질문을 광합성 관련 카테고리로 분류합니다.
- **L단계 프롬프트**: 학생 텍스트에서 핵심 키워드를 추출하여 SVG 모식도 코드를 생성합니다.
- **M단계 프롬프트**: 질문을 '개념 보충' 또는 '실생활 전이'로 분류하고 탐구 과제를 제안합니다.
- AI 응답은 항상 **중학교 2학년 수준**의 쉬운 언어로 작성되어야 합니다.

### 4-4. 보안 및 데이터 주의사항

- Claude API 키는 `index.html` 상단의 `CONFIG` 객체에서 교사가 직접 입력합니다. 코드에 하드코딩하지 않습니다.
- 학생 데이터는 `localStorage`에만 저장되며, 외부 서버로 전송되지 않습니다.
- 교사 대시보드 접근은 교사 코드(4자리 숫자) 입력으로 보호합니다.

### 4-5. 금지 사항

- ❌ React, Vue, Angular 등 프레임워크 사용 금지
- ❌ Node.js, Python 등 백엔드 서버 구축 금지
- ❌ npm, pip 등 패키지 설치 필요한 라이브러리 사용 금지
- ❌ 외부 데이터베이스 연동 금지 (localStorage만 사용)
- ❌ 영어 UI 텍스트 사용 금지 (모든 텍스트는 한국어)
